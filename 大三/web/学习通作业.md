### **第一章作业整理**

#### **一、填空题**

**1. B/S结构中的B指的是________（1）________，S指的是________（2）________。**

- **答案**： (1) 浏览器 (2) 服务器
- **知识点讲解**： **B/S结构**（Browser/Server，浏览器/服务器模式）是Web应用程序的经典架构。 **B（Browser）**：指的是**浏览器**，如Chrome、Edge等。用户通过浏览器来访问和使用应用，无需安装其他特定软件。这是该架构对用户最友好的特点。 **S（Server）**：指的是**服务器**，负责处理业务逻辑、数据存取等所有核心功能，并将结果生成网页返回给浏览器。

**2. Web系统的三层结构主要包括：（1）________（1）________、（2）________（2）________、（3）________（3）________。**

- **答案**： (1) 数据访问层 (2) 业务逻辑层 (3) 页面显示层
- **知识点讲解**： 三层架构是一种将应用程序的功能分为三个明确层次的设计模式，旨在实现“高内聚、低耦合”，便于开发和维护。 **数据访问层（DAL）**：**职责**：专门负责与数据库进行交互，执行数据的增、删、改、查（CRUD）操作。**比喻**：好比餐厅的后厨采购，只负责获取和存放食材。 **业务逻辑层（BLL）**：**职责**：包含应用程序的核心业务规则和逻辑。它从显示层接收请求，处理这些请求（如计算、验证、决策），并指示数据访问层进行相应的数据操作。**比喻**：好比餐厅的厨师，根据订单（请求）用食材（数据）烹饪菜肴（处理业务）。 **页面显示层（UIL）**：**职责**：负责向用户显示信息（UI）和接收用户输入。它不包含业务逻辑，只负责呈现和交互。在Web中，通常就是浏览器中看到的网页。**比喻**：好比餐厅的服务员和菜单，负责接待顾客（用户）和展示成品菜肴（数据）。

#### **二、简答题**

**3. 简述B/S结构最大的优点和C/S结构的主要缺点。**

- **答案**： **B/S结构最大的优点**：用户可以在任何地点、使用任何设备（如电脑、手机、平板）上的浏览器访问系统，**无需安装任何专用的客户端软件**，使用和维护都非常方便。 **C/S结构的主要缺点**： **只适用于局域网**：传统C/S结构通常部署在局域网内，远程访问需要复杂的配置（如VPN），可扩展性差。 **客户端需要安装专用的客户端软件**：每个用户都需要安装和配置客户端程序，部署和维护成本高。 **对客户端的操作系统一般也会有限制**：客户端软件通常针对特定操作系统（如Windows）开发，跨平台性差。
- **知识点讲解**： **C/S结构**（Client/Server，客户端/服务器模式）：优点在于能充分利用客户端计算机的性能，响应速度快。但其缺点在移动互联网时代愈发明显，主要体现在部署和跨平台方面。 **B/S结构**是C/S结构的一种变化和进步，它将客户端统一为**浏览器**，极大降低了客户端的维护成本，成为了当今互联网应用的主流架构。

### **第四章作业整理（ASP.NET内置对象）**

#### **一、单选题**

**1. 下列（ D ）对象不是保存在服务器中的。**

A. Application

B. Session

C. ViewState

D. Cookie

- **答案**：**D**
- **知识点讲解**： **Cookie**：数据存储在**客户端**（用户的浏览器中）。 **Application**, **Session**, **ViewState**：这些对象的数据都存储在**服务器端**的内存或特定状态服务器中。

**2. 下列（ C ）对象不能在页面间传送数据。**

A. Application

B. Session

C. ViewState

D. 查询字符串

- **答案**：**C**
- **知识点讲解**： **ViewState**：用于保存单个页面的状态，其生命周期仅限于当前页面。它会在页面回发时恢复状态，但**无法在不同页面之间传递数据**。 **Application**：在所有用户和所有页面间共享数据。 **Session**：在同一用户会话的不同页面间共享数据。 **查询字符串 (QueryString)**：通过URL的参数在不同页面间传递数据。

**3. 下面对Application对象描述正确的是（ A ）。**

A. Application对象可以共享信息

B. Application对象可以向页面中输出数据

C. Application对象能够获取IP地址

D. Application对象能够获取本地计算机的时间

- **答案**：**A**
- **知识点讲解**： **Application** 对象的核心作用是在整个Web应用程序范围内为**所有用户共享信息**（如网站访问计数器）。 向页面输出数据是 `Response`对象的功能。 获取IP地址是 `Request`对象的功能（`Request.UserHostAddress`）。 获取服务器时间是 `DateTime.Now`。

**4. 下面的选项中，哪个是通过Response对象向页面输出内容（ A ）。**

A. Response.Write

B. Response.Redirect

C. Response.WriteFile

D. Response.Flush

- **答案**：**A**
- **知识点讲解**： `Response.Write()`方法用于将指定的内容直接写入HTTP响应输出流，即输出到客户端浏览器。 `Response.Redirect()`用于将用户重定向到另一个URL。 `Response.WriteFile()`用于将指定的文件写入输出流。 `Response.Flush()`用于立即将缓冲区的所有输出发送到客户端。

**5. 要保存在应用程序的生命周期内使用的数据，需要使用（ B ）对象。**

A. Session

B. Application

C. Cookie

D. Server

- **答案**：**B**
- **知识点讲解**： **Application** 对象的生命周期与应用程序相同（从应用程序启动到终止），适合存储需要被所有用户长期使用的全局数据。 **Session** 对象的生命周期是用户会话期间。 **Cookie** 可以长期保存在客户端，但数据在客户端，不安全。 **Server** 对象提供服务器相关的方法属性，不是用于数据存储的。

**6. 下面的哪个对象可用于使服务器获取从客户端浏览器提交或上传的信息？（ A ）。**

A. Request

B. Response

C. Server

D. Cookie

- **答案**：**A**
- **知识点讲解**： **Request** 对象用于读取客户端在HTTP请求中发送的信息，包括： 表单数据 (`Request.Form`) 查询字符串参数 (`Request.QueryString`) 客户端浏览器信息 (`Request.Browser`) 客户端IP地址 (`Request.UserHostAddress`) 上传的文件 (`Request.Files`)

**7. 下面不属于Request对象能力范围内的是（ C ）。**

A. 获取客户端IP地址

B. 从URL中取回送来的参数

C. 向页面中输出数据

D. 获取浏览器信息

- **答案**：**C**
- **知识点讲解**： `Request`对象用于**获取**客户端信息。 `Response`对象用于**向页面中输出数据**。因此，“向页面中输出数据”不属于 `Request`对象的能力范围。

**8. 以下几个类中，不属于System.Data.OleDb命名空间的是（ C ）。**

A. OleDbConnection

B. DataSet

C. OleDbDataListReader

D. OleDbCommand

- **答案**：**C**
- **知识点讲解**： `System.Data.OleDb`命名空间包含用于连接和操作OLE DB数据源（如Access）的类，如 `OleDbConnection`, `OleDbCommand`, `OleDbDataReader`, `OleDbDataAdapter`。 **DataSet** 属于更上层的 `System.Data`命名空间，它是一个内存中的数据缓存，独立于任何特定数据库。 **OleDbDataListReader** 不是一个真实的类，标准的类是 `OleDbDataReader`。

#### **二、简答题**

**9. Session对象、Cookies对象和Application对象有哪些不同之处？**

- **答案要点与知识点讲解**： 这三个对象是ASP.NET中用于在不同范围和时间段内存储数据的机制，它们的核心区别体现在**作用范围**、**存储位置**和**生命周期**上。

| 对比维度     | **Application** 对象                                         | **Session** 对象                                             | **Cookies** 对象                                             |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **作用范围** | **全局应用程序级**                                           | **单个用户会话级**                                           | **单个用户客户端级**                                         |
|              | 对所有用户、所有页面共享。                                   | 在同一用户的不同页面间共享。                                 | 数据存储在用户浏览器上，可跨不同会话。                       |
| **存储位置** | **服务器内存**                                               | **服务器内存**（或状态服务器/数据库）                        | **客户端硬盘**或浏览器内存                                   |
| **生命周期** | 从**应用程序启动**到**应用程序结束**（如IIS重启）。          | 从**会话开始**（用户首次访问）到**会话结束**（超时或手动结束）。 | 可设置**过期时间**，从浏览器关闭即失效到永久保存。           |
| **主要用途** | 存储需要被**所有用户共享**的全局数据，如网站计数器、应用程序配置等。 | 存储**单个用户**的会话特定信息，如用户登录状态、购物车内容等。 | 在客户端存储少量非敏感信息，如用户偏好、用户名（用于下次自动登录）等。 |
| **安全性**   | 数据在服务器端，**相对安全**。                               | 数据在服务器端，**相对安全**。                               | 数据在客户端，**可能被用户查看或修改**，安全性较低。         |
| **性能影响** | 存储过多数据会占用**服务器内存**。                           | 为每个用户创建会话，**用户量巨大时会占用大量服务器资源**。   | **减轻服务器负担**，但每次HTTP请求都会携带Cookie数据，**增加网络流量**。 |
